# Guía de Contribución

¡Gracias por tu interés en contribuir a este repositorio!

## Reporte de errores (Issues)

Si detectas un error en un script, modelo o reporte:

1. Abre una nueva **Issue**.
2. Usa un título claro y específico.
3. Describe el problema incluyendo:
   - módulo o archivo afectado,
   - comportamiento esperado,
   - comportamiento observado,
   - pasos para reproducirlo,
   - versión de R y paquetes relevantes (si aplica).
4. Si puedes, adjunta un ejemplo reproducible y mensajes de error completos.

## Propuesta de nuevos módulos o scripts (Pull Requests)

Para proponer una mejora o un nuevo módulo estadístico:

1. Abre primero una **Issue** breve para discutir la propuesta y su alcance.
2. Crea una rama desde la rama principal con un nombre descriptivo.
3. Implementa los cambios manteniendo estructura y estilo del repositorio.
4. Incluye una explicación clara de:
   - objetivo del nuevo script/módulo,
   - método estadístico aplicado,
   - supuestos y limitaciones,
   - paquetes requeridos.
5. Envía un **Pull Request** con:
   - resumen de cambios,
   - contexto científico/técnico,
   - evidencia de funcionamiento (salidas, gráficos o validación).

## Buenas prácticas

- Mantener los cambios enfocados en un solo objetivo por PR.
- Usar nombres de archivos y carpetas consistentes con la organización actual.
- Evitar subir archivos temporales o de sesión local.

Gracias por ayudar a mantener un repositorio reproducible, claro y útil para la comunidad científica.
